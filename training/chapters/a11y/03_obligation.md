# Directive européenne 2016/2102

* Adoptée le **26 octobre 2016** par le Parlement européen
* Impose l'accessibilité des sites web et applications mobiles du secteur public
* Basée sur la norme **EN 301 549** (qui référence WCAG 2.1 niveau AA)
* Obligation de conformité progressive :
  - Nouveaux sites : septembre 2019
  - Sites existants : septembre 2020
  - Applications mobiles : juin 2021

**Source** : [Directive EU 2016/2102](https://eur-lex.europa.eu/eli/dir/2016/2102/oj)

---

# Obligations légales

* La directive européenne a été transposée dans des lois nationales :
    * **Public Sector Bodies Accessibility Regulations 2018** (UK)
    * **Décret du 25 juillet 2019** (France)
    * Chaque pays membre de l'UE a sa propre transposition

---

# Obligations légales

* Le secteur publique est composé de :
    * Autorités régionale, nationale et locale
    * Toutes organisations recevant des fonds publiques.

# Obligation legales

* Cette directive sera ensuite transposée dans les lois nationales :
    * Nouveau site web doit être accessible avant le 23/09/2019
    * Tout autre site web doit être accessible avant le 23/10/2020
    * Toutes les application mobile doit être accessible avant le 23/06/2021

# RGAA 4.1 - Référentiel français

* **RGAA** : Référentiel Général d'Amélioration de l'Accessibilité
* Version actuelle : **RGAA 4.1** (février 2021)
* Méthode technique française pour vérifier la conformité WCAG 2.1 niveau AA
* **106 critères** organisés en 13 thématiques
* Directement aligné avec WCAG 2.1 niveau AA + spécificités françaises

**Source** : [RGAA 4.1 officiel](https://accessibilite.numerique.gouv.fr/)

---

# RGAA 4.1 - Les 13 thématiques

1. Images
2. Cadres
3. Couleurs
4. Multimédia
5. Tableaux
6. Liens
7. Scripts
8. Éléments obligatoires
9. Structuration de l'information
10. Présentation de l'information
11. Formulaires
12. Navigation
13. Consultation

# En France - Secteur public

* Suite au décret du 25 juillet 2019, obligation pour :
  - Administrations publiques
  - Collectivités territoriales
  - Établissements publics
* **Sanctions possibles** : amende jusqu'à **20 000 € par an**

**Source** : [Décret n° 2019-768](https://www.legifrance.gouv.fr/eli/decret/2019/7/24/CPAE1913344D/jo/texte)

---

# En France - Secteur privé

* Extension au secteur privé depuis 2019
* Concerne les **entreprises de plus de 250 millions € de CA**
* S'applique aux services de communication en ligne
* Même niveau d'exigence : conformité RGAA 4.1 (WCAG 2.1 AA)

---

# En France - Documents obligatoires

En plus des correctifs techniques, 3 documents obligatoires :

1. **Déclaration d'accessibilité**
   - Niveau de conformité (non conforme, partiellement, totalement)
   - Résultats des tests
   - Justification des non-conformités

2. **Schéma pluriannuel** (3 ans)
   - Stratégie de mise en accessibilité

3. **Plans d'action annuels**
   - Actions concrètes issues du schéma pluriannuel

**Source** : [Obligations légales RGAA](https://accessibilite.numerique.gouv.fr/obligations/)

# Historique WCAG

* **WCAG 1.0** (1999) : Première version
* **WCAG 2.0** (2008) : Refonte majeure, devient ISO 40500
* **WCAG 2.1** (2018) : Ajout de 17 critères (mobile, basse vision, cognitif)
* **WCAG 2.2** (2023) : Ajout de 9 nouveaux critères
  - Focus Appearance (2.4.11, 2.4.12)
  - Dragging Movements (2.5.7)
  - Target Size (2.5.8)
  - Accessible Authentication (3.3.7, 3.3.8, 3.3.9)

**Source** : [W3C WCAG History](https://www.w3.org/WAI/standards-guidelines/wcag/)

---

# WCAG 2.2 - Structure

* **4 principes** (POUR)
* **13 guidelines** (lignes directrices)
* **86 critères testables** (success criteria)
* **3 niveaux de conformité** : A, AA et AAA

---

# Niveaux de conformité WCAG

## Niveau A (Minimum)

* **25 critères de base** essentiels
* Sans eux, l'accès au contenu est impossible pour certains utilisateurs
* Exemple : Toutes les images doivent avoir un texte alternatif

---

# Niveaux de conformité WCAG

## Niveau AA (Requis)

* **20 critères supplémentaires** (45 au total avec A)
* C'est le **niveau légalement requis** dans la plupart des pays
* Exemple : Ratio de contraste minimum 4.5:1 pour le texte normal

---

# Niveaux de conformité WCAG

## Niveau AAA (Excellence)

* **21 critères supplémentaires** (66 au total)
* Le plus haut niveau de conformité
* Recommandé mais **pas toujours atteignable** pour tout le contenu
* Exemple : Ratio de contraste 7:1, langue des signes pour vidéos

**Source** : [Understanding Levels of Conformance](https://www.w3.org/WAI/WCAG22/Understanding/conformance#levels)

# Cas de condamnations - Domino's Pizza (USA)

* **2019** : Condamnation par la Cour suprême américaine
* Plainte déposée par un utilisateur aveugle
* Site web et application mobile inaccessibles
* Impossible de commander en ligne avec un lecteur d'écran
* **Coût** : Frais juridiques + refonte complète du site
* Jurisprudence importante : les sites web = "lieux publics"

**Source** : [Robles v. Domino's Pizza](https://www.supremecourt.gov/search.aspx?filename=/docket/docketfiles/html/public/18-1539.html)

---

# Cas de condamnations - Beyoncé (USA)

* **2019** : Poursuite judiciaire contre le site parkwood-ent.com
* Site de merchandising de Beyoncé non conforme ADA
* Images sans texte alternatif
* Navigation impossible au clavier
* **Règlement à l'amiable** + mise en conformité du site

**Source** : [Billboard Article](https://www.billboard.com/business/legal/beyonce-sued-website-violating-americans-disabilities-act-8490493/)

# Cas de condamnations - France

# Success Story - GOV.UK

* Site du gouvernement britannique
* **Reconçu en 2012** avec l'accessibilité comme priorité #1
* Conformité WCAG 2.1 niveau AA
* **Résultats** :
  - 60% de réduction des appels au support
  - Satisfaction utilisateur passée de 62% à 85%
  - Prix "Design of the Year" 2013
  - Économies estimées : £60-80M par an

**Source** : [GOV.UK Accessibility Blog](https://accessibility.blog.gov.uk/)

---

# Success Story - Microsoft

* Programme d'accessibilité depuis les années 2000
* **Xbox Adaptive Controller** (2018) : manette adaptée
* Sous-titres automatiques dans Teams
* Narrator (lecteur d'écran) intégré à Windows
* **Impact** :
  - Marché élargi (1 milliard de personnes handicapées)
  - Image de marque positive
  - Innovation technologique bénéficiant à tous

**Source** : [Microsoft Accessibility](https://www.microsoft.com/en-us/accessibility)

# Success Story - BBC

# Calculer le ROI de l'accessibilité

## Investissement

- **Audit initial** : 1-2 semaines
- **Formation équipe** : 1-2 jours
- **Implémentation** : 5-10% du temps de développement
- **Tests continus** : Intégré dans la QA

## Retours

- **Audience** : +15% (1+ milliard de personnes)
- **SEO** : +30% de visibilité organique
- **Support** : -40% de réclamations
- **Protection légale** : Éviter les procès (coût moyen : 50K-500K€)
- **Réputation** : Relations publiques positives
